# Istall CUDA and cuDNN

## What Are Those?
For those of you not familiar - these are basically the drivers required for using GPUs (*Graphical Processing Units* - sometimes in this context also called *GPGPUs* where the other *GP* stands for *General Purpose*). Installing them is a lot of fun.

## Installing CUDA


Let's first make sure we don't have an older version installed by hitting `sudo apt-get purge nvidia cuda `

Also do this just in case:
```
sudo apt-get autoremove 
sudo dpkg -P cuda-repo-ubuntu1604
```

Then finally install:

```
sudo apt-get autoremove && \
sudo apt-get update -y && \
sudo apt-get install -y build-essential git gcc-multilib && \
sudo apt-get install -y linux-headers-$(uname -r)
```

Now the actual drivers:

```
sudo add-apt-repository -y ppa:graphics-drivers/ppa && \
sudo apt update -y && \
sudo apt install -y nvidia-390
```

Let's check that this worked:

```
nvidia-smi -q | head && ls -la /dev | grep nvidia
```

### (Conditional) UVM Error Handling:

If we get and error along the lines of "ERROR: The nvidia-uvm kernel module was not created.", do this:

(This happened with 384 and I believe was fixed for 390)

```
sudo ./NVIDIA-Linux-x86_64-*.run -x && \
sudo sed -i '1s/^/ #include <linux\/sched\/task_stack.h> \n /' ./NVIDIA-Linux-x86_64-*/kernel/nvidia-uvm/uvm8_va_block.c && \
sudo ./NVIDIA-Linux-x86_64-384.81/nvidia-installer -s
```

## Install cuDNN

### Register and Download

*cuDNN*, unlike *CUDA*, requires us to register with NVIDIA's developer center. Let's do that by going [here](https://developer.nvidia.com/rdp/cudnn-download). Once done, download the correct *cuDNN* distribution. Note that for this guide, we want to get version *7.0.5* and nothing earlier or later. Archived versions are in [this](https://developer.nvidia.com/rdp/cudnn-archive) link.

We want to download **Download cuDNN v7.0.5 (Dec 5, 2017), for CUDA 9.0**. [This](https://developer.nvidia.com/rdp/cudnn-archive#a-collapse705-9) might work directly, again. The file we want will always be called **cuDNN v7.0.5 Library for Linux without (Deb)**. Download it and place in your homedir inside the machine.

### Now Install

Let's:

```
sudo tar xvf cudnn-9.0-linux-x64-v7.tgz --strip-components=1 -C /usr/local/cuda-9.0/
```

### Anaconda

Starting Assignment 2 we believe it's better to use Anaconda for managing python environments and packages. For assignment one we used a simple *virtualenv* but it's not a recommended option for heavier projects.

We direct you to the Anaconda website, where you can follow instructions to get your system up and running. After done, we recommend installing the PyTorch *or* Tensorflow using the Anaconda installation method. 

Once anaconda is actually installed, we can create a new environment using:

```
conda create -n cs231n python=3.6 anaconda
```

To *activate* or *exit/deactivate* an environment, we will use `source activate cs231n` and `deactivate`, respectively. 

To install a specific TF or PyTorch version we can specify them in the installer once inside an environment. I'd use:

```
conda install -c pytorch pytorch=0.4.1
```

Note that the first `pytorch` directs Anaconda to use a specific *channel*.

For TF, use:

```
conda install -c conda-forge tensorflow=1.8.0
```
